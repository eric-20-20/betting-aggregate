# Review Slate Schema

Schema specification for the editorial review workflow files.

## File Locations

All review artifacts use a dated directory layout:

```
data/review/{sport}/{YYYY-MM-DD}/
    review_slate.json           Draft review dataset
    review_slate.csv            Spreadsheet-friendly view
    overrides.json              Manual override file (optional)
    approved_slate.json         Resolved approved slate

Example:
    data/review/nba/2026-04-20/review_slate.json
    data/review/ncaab/2026-04-20/overrides.json
```

## Slate File (slate_YYYY-MM-DD.json)

Generated by `scripts/generate_review_slate.py`. Contains all candidate picks for editorial review.

```json
{
  "meta": {
    "date": "2026-04-20",
    "sport": "NBA",
    "generated_at_utc": "2026-04-20T15:30:00+00:00",
    "source_plays_file": "data/plays/plays_2026-04-20.json",
    "total_candidates": 45,
    "auto_included_count": 42,
    "near_miss_count": 3
  },
  "candidates": [ ...review rows... ]
}
```

### Candidate Row Fields

| Field | Type | Description |
|-------|------|-------------|
| `pick_id` | string | Unique ID (= signal_id from pipeline, SHA-256 hash) |
| `date` | string | Game date (YYYY-MM-DD) |
| `sport` | string | "NBA" or "NCAAB" |
| `game_label` | string | "ATL @ BOS" |
| `bet_label` | string | Human-readable: "LeBron James OVER 25.5 points" |
| `market` | string | market_type: player_prop, spread, total, moneyline |
| `selection` | string | Raw selection key: "NBA:lebron_james::points::OVER" |
| `line` | float/null | Betting line (e.g., 25.5) |
| `odds` | object/null | Odds data from pipeline |
| `auto_tier` | string | Pipeline-assigned tier: A, B, C, or D |
| `tier_reason` | string | Human-readable explanation of tier decision |
| `support_count` | int | Number of agreeing sources |
| `pattern_name` | string/null | Matched pattern label, if any |
| `source_list` | string[] | Sources present (internal, not exported to web) |
| `auto_included` | bool | True if auto_tier in (A, B, C) |
| `website_eligible` | bool | True if passed quality gate |
| `published_tier` | string/null | Null until override applied |
| `published_included` | bool/null | Null until override applied |
| `override_note` | string/null | Null until override applied |
| `wilson_score` | float | Wilson lower bound score |
| `confidence_score` | int | 0-100 confidence scale |
| `gate_reason` | string/null | Why filtered (D-tier only) |
| `primary_record` | object | Historical win/loss record backing this pick |
| `matched_pattern` | object/null | Pattern match details |
| `_play_data` | object/null | Full play dict (for D-tier near-misses that may be promoted) |

### tier_reason Examples

- `"A: pattern 'covers+dimers / spread' (62-34, Wilson 0.590) + 4 sources"`
- `"B: Wilson 0.48, 2 sources, no pattern match"`
- `"C: marginal Wilson 0.46, 2 sources"`
- `"D(near-miss): Wilson 0.43, gated -- quality_gate:low_wilson"`

## Override File (overrides_YYYY-MM-DD.json)

Manually created by the editor. Only include picks you want to change.

```json
{
  "overrides": [
    {
      "pick_id": "abc123...",
      "published_tier": "B",
      "published_included": true,
      "override_note": "Promoting -- line movement confirmed"
    },
    {
      "pick_id": "def456...",
      "published_included": false,
      "override_note": "Key player ruled out"
    }
  ]
}
```

### Override Entry Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `pick_id` | string | Yes | Must match a pick_id in the slate |
| `published_tier` | string | No | Override tier: A, B, C, or D |
| `published_included` | bool | No | Override inclusion on website |
| `override_note` | string | No | Reason for override |

### Validation Rules

- `pick_id` must exist in the slate
- `published_tier` must be in {A, B, C, D}
- `published_included` must be boolean
- No duplicate pick_ids
- No unknown fields
- Script exits non-zero on any validation failure

### Fallback Behavior

- If no override file exists: published = auto for all picks
- If override exists but omits a pick: that pick falls back to auto
- If override sets `published_tier` but not `published_included`: included falls back to auto (based on new tier)

## Approved Slate (approved_YYYY-MM-DD.json)

Generated by `scripts/apply_overrides.py`. Same schema as slate file, but all `published_tier` and `published_included` fields are resolved (never null).

```json
{
  "meta": {
    "date": "2026-04-20",
    "sport": "NBA",
    "generated_at_utc": "...",
    "source_slate_file": "data/review/slate_2026-04-20.json",
    "override_file": "data/review/overrides_2026-04-20.json",
    "override_count": 2,
    "total_candidates": 45,
    "published_included_count": 41,
    "published_excluded_count": 4,
    "overrides_applied": 2,
    "auto_fallback_count": 43
  },
  "candidates": [ ...resolved review rows... ]
}
```

## CSV Format (slate_YYYY-MM-DD.csv)

Flat CSV with one row per candidate. Complex fields (primary_record, matched_pattern, odds) are omitted. Columns:

```
pick_id, date, sport, game_label, bet_label, market, selection, line,
auto_tier, tier_reason, support_count, pattern_name, source_list,
auto_included, wilson_score, confidence_score, gate_reason
```

Designed for quick spreadsheet review. Use the JSON file for programmatic access.
